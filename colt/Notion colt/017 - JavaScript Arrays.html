178. Introducing Arrays

mixed arrays in js

In JavaScript, an array is a data structure that allows you to store multiple values in a single variable. Normally, arrays in JavaScript can contain elements of the same type, such as numbers, strings, or objects. However, JavaScript also allows you to create mixed arrays that can contain different types of values, including numbers, strings, objects, or even other arrays.

Here's an example of a mixed array in JavaScript:

let mixedArray = [42, "Hello", {name: "John"}, true];

In the above example, the mixedArray contains four elements of different types:

  1. 42 is a number.
  1. "Hello" is a string.
  1. {name: "John"} is an object literal.
  1. true is a boolean value.

You can access individual elements of a mixed array using their index. In JavaScript, arrays are zero-indexed, meaning the first element has an index of 0, the second element has an index of 1, and so on. Here's how you can access the elements of the mixedArray:

console.log(mixedArray[0]); // Output: 42
console.log(mixedArray[1]); // Output: "Hello"
console.log(mixedArray[2]); // Output: {name: "John"}
console.log(mixedArray[3]); // Output: true

You can also modify elements of a mixed array or add new elements to it using assignment:

mixedArray[1] = "World";
mixedArray[4] = 3.14;

console.log(mixedArray); // Output: [42, "World", {name: "John"}, true, 3.14]

It's worth noting that using mixed arrays with different types can sometimes make code harder to read and maintain. It's generally a good practice to use arrays to store elements of the same type or related data. However, in certain situations, mixed arrays can be useful, especially when dealing with data structures where elements have varying types.

179. Array Random Access

let colours = ["red","blue","green"] and colours[10] = "yellow" how this is working in js

In JavaScript, arrays are dynamic data structures, meaning you can add or modify elements even if they go beyond the initial length of the array.

In the example you provided:

var colours = ["red", "blue", "green"];
colours[10] = "yellow";

The colours array is initially defined with three elements: "red", "blue", and "green". However, when you assign a value to colours[10], JavaScript automatically adjusts the size of the array to accommodate the new element at the specified index. In this case, JavaScript will add seven empty elements between the existing elements and the new element.

If you were to output the colours array after adding the new element, you would get the following:

console.log(colours);
// Output: ["red", "blue", "green", empty x 7, "yellow"]

The output shows that the array now has a length of 11, and there are empty elements (undefined) at indices 3 to 9. Finally, at index 10, you have the value "yellow".

It's important to note that while JavaScript allows you to add or modify elements beyond the array's original length, it doesn't automatically fill the gaps with meaningful values. In this case, JavaScript simply adds empty elements (undefined) to the array. If you were to access colours[4], for example, you would get undefined because there is no value assigned at that index.

185. Arrays + Const

means we cannot create the variable array again with in the scope

In JavaScript, the const keyword is used to declare a constant variable that cannot be reassigned once it has been assigned a value. This applies to individual variables as well as arrays. While you cannot change the reference of a const variable, you can modify the contents of the variable or array.

Let's look at some examples:

  1. Const assignment to individual variables:
const pi = 3.14159;
console.log(pi); // Output: 3.14159

pi = 3.14; // Error: Assignment to constant variable

In this example, we declare a constant variable pi and assign it the value of 3.14159. When we try to reassign a new value to pi, it throws an error because const variables cannot be reassigned.

  1. Const assignment to arrays:
const numbers = [1, 2, 3, 4, 5];
console.log(numbers); // Output: [1, 2, 3, 4, 5]

numbers.push(6);
console.log(numbers); // Output: [1, 2, 3, 4, 5, 6]

numbers = [7, 8, 9]; // Error: Assignment to constant variable

In this example, we declare a constant variable numbers and assign an array [1, 2, 3, 4, 5] to it. Although we cannot reassign a new array to numbers, we can modify the array itself by using methods like push(). As you can see, we successfully added the number 6 to the array. However, when we attempt to assign a new array [7, 8, 9] to numbers, it throws an error because the reference of a const array cannot be changed.

To summarize, const in JavaScript allows you to assign a value to a variable or an array that cannot be reassigned to a new value, but you can still modify the contents of the variable or array within certain constraints.

const numbers = [5, 3, 1, 4, 2];
numbers.sort();
console.log(numbers); // Output: [1, 2, 3, 4, 5]

In this example, we have an array numbers with unsorted elements. We call the sort() method on the numbers array, and it sorts the elements in ascending order. The sorted array is then printed to the console.

  1. Lead-in Concept: Const does not allow for reassignment, ensuring constant values like Pi remain unchanged.
  1. Arrays and References: Variables that store arrays hold references to their addresses rather than the contents.
  1. Using const with Arrays: Const can be used with arrays, allowing changes to the array's contents but not the reference.
  1. Content Modification: Modifying array contents, such as adding, deleting, or changing elements, is allowed with const.
  1. Reassigning Arrays: Reassigning an array to a different reference or a different array results in an error.
  1. Importance of Container Stability: Using const with arrays ensures that the container's reference remains the same while allowing content changes.
  1. Safer Array Handling: Using const with arrays provides a safer way of working, especially when maintaining specific structures or queues.
  1. Array Behavior vs. Primitive Variables: Arrays behave differently with const compared to primitive variables, allowing content modifications but not reassignment.
  1. Objects and References: Similar behavior applies to objects, where the reference cannot be reassigned but the object's properties can be modified.
