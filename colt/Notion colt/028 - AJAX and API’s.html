296. Making XHRs

XML HTTP requests and why they are considered outdated:

  1. Older Approach - XML HTTP Request (XHR): XHR is highlighted as the older and less useful way of making HTTP requests from JavaScript.
  1. Introduction for Context: The demonstration of XHR is presented to give context and set the stage for the advancements in modern approaches like Fetch and Axios.
  1. Lack of Promise Support: XHR lacks support for promises, resulting in a reliance on callbacks for handling requests, making the code structure cumbersome and messy.
  1. Syntax and Capitalization Issues: The syntax of XHR is described as clunky and difficult to remember. The inconsistent capitalization in terms like XML and HTTP adds to the complexity.
  1. Preparation for Learning Modern Approaches: While XHR is presented for understanding, it's emphasized that memorizing its syntax is unnecessary, preparing the learner for the subsequent introduction of more efficient methods like Fetch and Axios.

In summary, the drawbacks of the XHR approach, including its callback-based nature, syntax challenges, and lack of promise support, while also setting the stage for learning newer and better alternatives like Fetch and Axios.

How to Make a request

Making a request using XML HTTP request and preparing the response for use:

  1. XML HTTP Request (XHR): The original method for making HTTP requests from JavaScript. Involves creating a new XHR object, opening a connection with a specified method (e.g., GET), providing the URL, sending the request, and handling the response and potential errors.
  1. Creating XHR Object: To make a request, create an XHR object using new XMLHttpRequest() and save it to a variable (e.g., request).
  1. Opening the Request: Use request.open() to specify the method (e.g., GET) and the URL you want to request.
  1. Sending the Request: After opening the request, use request.send() to send the request.
  1. Handling Callbacks: Set up callbacks to handle the response. Typically, you'll use request.onload for successful responses and request.onerror for errors.
  1. Parsing Response: The response is initially in text format. To work with it as a JavaScript object, parse it using JSON.parse() to convert it into an object.
  1. Using the Response: Once parsed, you can access data in the response object and use it in your application.

The XHR approach is considered outdated and cumbersome compared to modern alternatives like fetch and Axios, which offer a cleaner and more efficient syntax.
