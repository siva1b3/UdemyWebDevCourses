269. Input & Change Events

  • input: any change made in the text content
  • change:
    • If it is an <input />: change + lose focus
    • If it is a <select>: change option

  • The change event fires in most browsers when content is changed and the element loses focus. It's basically an aggregate of changes. It will not fire for every single change as in the case input event.
  • The input event fires synchronously on change of the content for the element. As such, the event listener tends to fire more frequently.

When a browser loads a website containing a <script> tag, several steps occur to ensure proper rendering and execution of the webpage's content. The process involves parsing HTML, handling script downloads, and ensuring optimal user experience. Here's a detailed breakdown of what happens:

  1. Fetch the HTML Page: When you enter a URL or click on a link, the browser sends a request to the web server hosting the website. The server responds with the HTML file of the webpage, often referred to as "index.html."
  1. Parsing the HTML: Once the HTML file is received, the browser starts parsing it. Parsing is the process of analyzing the HTML code to create a structured representation of the webpage called the Document Object Model (DOM). The DOM is a hierarchical tree-like structure that represents the elements, attributes, and relationships within the HTML.
  1. Encounter a <script> Tag: During the parsing process, if the browser encounters a <script> tag in the HTML, it indicates that an external script file needs to be loaded and executed. The script tag contains the src attribute pointing to the location of the script file.
  1. Requesting the Script File: As soon as the browser identifies a <script> tag with the src attribute, it initiates a separate HTTP request to fetch the script file from the specified location. This request is made synchronously, allowing other tasks to wait while waiting for the script file to be downloaded.
  1. Parser Blocking: At this point, a critical aspect comes into play. The parsing of HTML is halted while the browser waits for the script file to be downloaded and executed. This is because the script could potentially manipulate the DOM using functions like document.write(), which would require the parser to wait until the script's effects are accounted for in the DOM structure.
  1. Script Download and Execution: The browser retrieves the script file from the server. Once the file is downloaded, the browser executes the JavaScript code within the script. This may involve actions like modifying the DOM, interacting with server APIs, or setting up event listeners.
  1. Resuming HTML Parsing: After the script has been executed, the browser resumes parsing the HTML document from where it left off. It continues building the DOM tree, processing other elements, and rendering content.
  1. Modern Approaches to Script Loading: In the past, developers used to place <script> tags at the bottom of the <body> element to prevent parser blocking. However, this approach delayed script execution and hindered performance. Modern solutions have emerged to address this issue:
    • Async Attribute (async): Adding the async attribute to a <script> tag indicates that the script can be fetched asynchronously while allowing HTML parsing to continue. Scripts with the async attribute are executed as soon as they're downloaded, potentially out of order.
    • Defer Attribute (defer): Adding the defer attribute to a <script> tag also allows asynchronous loading of the script. However, scripts with the defer attribute are executed in the order they appear in the HTML. Importantly, they are executed only after the entire HTML document has been parsed.
  1. Choosing Async or Defer: Developers can choose between the async and defer attributes based on their requirements. If scripts need to be executed in a specific order and depend on the DOM, defer is a better choice. On the other hand, if scripts are independent and can run concurrently, async can improve page load times.
  1. Optimal Solution for Performance: The current best practice is to place scripts in the <head> tag of the HTML and use the async or defer attributes. This approach enables scripts to start downloading as soon as possible without blocking the browser's rendering of the webpage. This strategy balances the need for timely script execution with maintaining a smooth user experience.

By adopting these modern loading techniques, websites can provide faster loading times and improved user satisfaction, while still accommodating the small percentage of browsers that might not support these attributes. This practice helps strike a balance between script execution and overall webpage rendering.
