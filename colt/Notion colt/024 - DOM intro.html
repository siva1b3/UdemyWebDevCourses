Explanation of DOM

Document Object Model (DOM) is an integral part of web development and it is essential for interacting with the contents of a web page using JavaScript.

Here's a breakdown of the key points:

  1. The DOM is an object: The DOM is a programming interface that represents a web page's structure and content as objects. These objects can be manipulated using JavaScript to modify the page dynamically.
  1. Created automatically by the browser: When a web page is loaded in a browser, the DOM is automatically created based on the HTML structure of the page. It parses the HTML and creates a tree-like structure, where each element in the HTML is represented as a node in the DOM tree.
  1. Bunch of objects: The DOM consists of various objects that represent different elements on the web page. For example, each HTML element like headings, paragraphs, images, etc., is represented as an object in the DOM.
  1. Methods and properties: The DOM provides a set of methods and properties that developers can use to interact with and manipulate the content and behavior of the web page. For instance, developers can use DOM methods to add event listeners to elements, change CSS styles dynamically, or hide/show elements on the page.
  1. Top-level folder: The DOM acts as the top-level container for all the objects and elements on the web page. It serves as the entry point to access and modify the entire content of the page.
  1. Root of the tree: In computer science terms, the DOM is often compared to a tree data structure, where each element is a node, and the top-most node is known as the root of the tree. The DOM tree represents the hierarchical structure of the HTML document.

Overall, the DOM plays a crucial role in web development as it allows developers to interact with the web page dynamically and build responsive and interactive web applications. By accessing and manipulating the DOM, developers can create rich user experiences and make their websites more dynamic and engaging.

When you create an object or instance of a class in JavaScript, it can inherit properties and methods from its prototype. The prototype is essentially an object that acts as a blueprint for other objects to follow. Inheritance allows objects to share common properties and methods, reducing redundancy and promoting code reusability.

In JavaScript, some properties are read-only, meaning you cannot directly assign new values to them. The currentSrc property of an HTMLImageElement object is one such property that is read-only.

The currentSrc property represents the absolute URL of the image that is currently being displayed in the image element. It is automatically updated by the browser to reflect the actual source URL of the image that is loaded. When you try to set a new value to banner.currentSrc, the assignment is simply ignored by the browser, and the property remains unchanged.

This behavior is by design to prevent developers from mistakenly modifying or tampering with the source URL of an image that is currently being displayed. The src property of an HTMLImageElement is the property you should use if you want to change the image displayed, but even then, you should be careful with updating the src attribute dynamically, as it could lead to unintended behavior or loading issues.

251. Attributes

so let's compare link.getAttribute("href") with link.href for retrieving the value of the "href" attribute of an anchor element (<a>).

  1. link.getAttribute("href"):
    • This method is a generic way to retrieve any attribute of an HTML element, including the "href" attribute of an anchor element.
    • It works for any attribute and not just predefined properties.
    • It returns the attribute value as a string.

    Example usage:

    const link = document.getElementById("myLink");
const hrefValue = link.getAttribute("href");
console.log(hrefValue); // Output: the value of the "href" attribute as a string
  1. link.href:
    • This property is a direct DOM property that corresponds specifically to the "href" attribute of an anchor element (<a>).
    • It provides a convenient way to directly access the value of the "href" attribute without using getAttribute.
    • When you access link.href, it returns the absolute URL as a string.
    • If the "href" attribute is relative (e.g., "page.html" or "#section"), link.href will return the resolved absolute URL.

    Example usage:

    const link = document.getElementById("myLink");
const hrefValue = link.href;
console.log(hrefValue); // Output: the absolute URL as a string

In most cases, you can use link.href to access the "href" attribute, as it's more concise and specific to anchor elements. However, if you need to retrieve a custom attribute or any attribute that is not natively exposed as a property in the DOM, getAttribute would be the appropriate method to use.
